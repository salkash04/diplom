Введение в обработку HTTP-запросов

В этом разделе мы рассмотрим, как обрабатывать различные типы HTTP-запросов и управлять кодами ответа в Spring.

1. Обработка GET-запросов

GET-запросы используются для получения данных с сервера. В Spring вы можете обрабатывать их с помощью аннотации @GetMapping.

Пример использования:

@GetMapping("/users/{id}")
public ResponseEntity<User> getUser(@PathVariable Long id) {
    User user = userService.findById(id);
    return ResponseEntity.ok(user); // Код ответа 200
}

2. Обработка POST-запросов

POST-запросы используются для создания новых ресурсов на сервере. В Spring вы можете обрабатывать их с помощью аннотации @PostMapping.

Пример использования:

@PostMapping("/users")
public ResponseEntity<User> createUser(@RequestBody User user) {
    User createdUser = userService.save(user);
    return ResponseEntity.status(HttpStatus.CREATED).body(createdUser); // Код ответа 201
}

3. Обработка PUT-запросов

PUT-запросы используются для обновления существующих ресурсов. В Spring вы можете обрабатывать их с помощью аннотации @PutMapping.

Пример использования:

@PutMapping("/users/{id}")
public ResponseEntity<User> updateUser(@PathVariable Long id, @RequestBody User user) {
    User updatedUser = userService.update(id, user);
    return ResponseEntity.ok(updatedUser); // Код ответа 200
}

4. Обработка DELETE-запросов

DELETE-запросы используются для удаления ресурсов на сервере. В Spring вы можете обрабатывать их с помощью аннотации @DeleteMapping.

Пример использования:

@DeleteMapping("/users/{id}")
public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
    userService.delete(id);
    return ResponseEntity.noContent().build(); // Код ответа 204
}

5. Управление кодами ответа HTTP

Spring позволяет управлять кодами ответа HTTP с помощью класса ResponseEntity. Вы можете указать статус и тело ответа в зависимости от результата обработки запросов.

Пример использования ResponseEntity:

@GetMapping("/error")
public ResponseEntity<String> handleError() {
    return new ResponseEntity<>("Ошибка обработки запроса", HttpStatus.INTERNAL_SERVER_ERROR); // Код ответа 500
}

Заключение

Теперь вы знаете, как обрабатывать различные типы HTTP-запросов и управлять кодами ответа в Spring. Это позволяет создавать полноценные RESTful API для взаимодействия с клиентами.
