Введение в запуск приложений на Spring Boot

В этом разделе мы рассмотрим, как запустить приложение Spring Boot и использовать встроенный сервер Tomcat.

Запуск приложения Spring Boot

Запуск через командную строку:

Вы можете запустить приложение Spring Boot через командную строку следующими командами:

  • Если вы используете Maven:
    • mvn spring-boot:run
  • Если вы используете Gradle:
    • ./gradlew bootRun
  • Если у вас есть JAR-файл:
    • java -jar ваше-приложение.jar

Запуск из IDE:

Вы также можете запустить приложение Spring Boot из вашей IDE как простое Java приложение. Для этого нужно импортировать ваш проект в IDE.

Подготовка к запуску

Перед запуском приложения убедитесь, что у вас установлены все необходимые зависимости в файле pom.xml (для Maven) или build.gradle (для Gradle).

Использование встроенного сервера Tomcat

Spring Boot включает встроенный сервер Tomcat по умолчанию, что позволяет вам запускать веб-приложения без необходимости устанавливать Tomcat отдельно. Это упрощает процесс разработки и тестирования.

Отладка приложения

Для отладки приложения вы можете использовать параметры JVM, чтобы подключить отладчик. Например:

java -Xdebug -Xrunjdwp:server=y,transport=dt_socket,address=8000,suspend=n -jar target/myapplication-0.0.1-SNAPSHOT.jar

Проверка работы приложения

После запуска приложения перейдите по адресу http://localhost:8080/hello (или другому маршруту, который вы определили) для проверки работы вашего RESTful API.

Заключение

Теперь вы знаете, как запустить приложение Spring Boot и работать с встроенным сервером Tomcat. Это позволяет быстро разрабатывать и тестировать ваши веб-приложения.
