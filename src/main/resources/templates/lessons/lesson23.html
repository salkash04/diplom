Введение в глобальную обработку ошибок

В этом разделе мы рассмотрим, как использовать аннотацию @ControllerAdvice для централизованной обработки ошибок в приложении Spring.

Что такое @ControllerAdvice?

@ControllerAdvice — это аннотация, которая позволяет объединить обработку исключений в одном месте и применять ее ко всем контроллерам приложения.

Обработка исключений с помощью @ExceptionHandler:

Вы можете использовать аннотацию @ExceptionHandler внутри класса, помеченного @ControllerAdvice, чтобы обрабатывать конкретные исключения.

Пример использования:

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<Response> handleBusinessException(BusinessException e) {
        Response response = new Response(e.getMessage());
        return new ResponseEntity<>(response, HttpStatus.BAD_REQUEST);
    }
}

Обработка нескольких исключений:

Вы можете обрабатывать несколько типов исключений в одном методе:

@ExceptionHandler({BusinessException.class, ServiceException.class})
public ResponseEntity<Response> handleMultipleExceptions(Exception e) {
    Response response = new Response(e.getMessage());
    return new ResponseEntity<>(response, HttpStatus.BAD_REQUEST);
}

Настройка глобального обработчика:

Вы можете настроить обработчик для работы только с определенными контроллерами, добавив соответствующие аннотации:

@ControllerAdvice(annotations = CustomAnnotation.class)
public class CustomAdvice {
    // Обработка исключений
}

Заключение

Использование @ControllerAdvice позволяет упростить управление ошибками в приложении и сделать код более чистым и поддерживаемым.
