Введение в файлы конфигурации Spring Boot

В этом разделе мы рассмотрим, как использовать файлы конфигурации application.properties и application.yml для настройки вашего приложения Spring Boot.

Файл application.properties

Файл application.properties — это стандартный файл конфигурации, используемый в Spring Boot для настройки различных параметров приложения.

Пример использования:

spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=password
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

Файл application.yml

Файл application.yml — это альтернативный формат для настройки приложения, который более удобен для чтения и структурирования данных.

Пример использования:

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: password
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

Выбор между properties и YAML

Выбор между использованием application.properties и application.yml зависит от ваших предпочтений. YAML позволяет более удобно структурировать данные, особенно если у вас много уровней вложенности.

Заключение

Теперь вы знаете, как использовать файлы конфигурации application.properties и application.yml в Spring Boot для настройки вашего приложения.
