Введение в юнит-тестирование

Юнит-тестирование — это метод тестирования программного обеспечения, при котором отдельные части приложения (юниты) проверяются на корректность работы.

1. Знакомство с JUnit

JUnit — это фреймворк для написания тестов на Java. Он предоставляет аннотации и методы для организации тестов.

Пример простого теста:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

2. Знакомство с Mockito

Mockito — это фреймворк для создания мок-объектов, который упрощает тестирование классов с внешними зависимостями.

Пример использования Mockito:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository userRepository = mock(UserRepository.class);
        UserService userService = new UserService(userRepository);

        User user = new User("John");
        when(userRepository.findById(1L)).thenReturn(user);

        User result = userService.getUser(1L);
        assertEquals("John", result.getName());

        verify(userRepository).findById(1L); // Проверка вызова метода
    }
}

3. Основные аннотации JUnit и Mockito:

  • @Test: Обозначает метод как тестовый.
  • @Mock: Создает мок-объект.
  • @InjectMocks: Внедряет моки в тестируемый класс.
  • @BeforeEach: Выполняет метод перед каждым тестом.
  • @AfterEach: Выполняет метод после каждого теста.

4. Запуск тестов

Тесты можно запускать из IDE или с помощью командной строки, используя Maven или Gradle.

Заключение

Теперь вы знаете, как писать юнит-тесты с использованием JUnit и Mockito. Эти инструменты помогают обеспечить качество кода и его корректность.
